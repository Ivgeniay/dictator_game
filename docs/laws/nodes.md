# Ноды языка законов

`Domain/Laws/Nodes/Types.cs`

Законопроект в игре составляется игроком из визуальных блоков - как в Scratch. Каждый блок это нода. Ноды - это минимальные единицы смысла из которых строится тезис закона.

## Зачем это нужно

В реальной жизни закон это текст на естественном языке. В игре нам нужна структура которую можно программно анализировать - понять кого затрагивает закон, что он запрещает и насколько строг. Ноды это и есть такая структура.

## Node

Абстрактный базовый класс для всех нод. Хранит поле `Type` - строковое имя конкретного класса. Используется при сериализации чтобы при восстановлении из JSON знать какой именно тип создавать.

## Типы нод

Ноды оперируют юридическими понятиями - `SubjectType`, `ActionType`, `RestrictionType`, `CircumstanceType`. Описание этих типов находится в разделе [Юридические понятия]({% link shared/legal_terms.md %}).

**`SubjectNode`** - нода содержащая один субъект. Аналог одного имени в предложении "запрещается гражданам и чиновникам".

**`SubjectGroupNode`** - материнская нода которая объединяет субъектов и логические операторы. Читается слева направо: все субъекты до `NOT` включаются в действие закона, после `NOT` - исключаются.

**`ActionNode`** - нода содержащая одно действие.

**`RestrictionNode`** - нода содержащая одно ограничение.

**`CircumstanceNode`** - нода содержащая одно обстоятельство.

**`CircumstanceGroupNode`** - материнская нода для обстоятельств. Работает по той же логике что `SubjectGroupNode`.

**`LogicOperator`** - перечисление логических операторов: AND, OR, NOT. Используется в группах для выражения "чиновники AND СМИ" или "все NOT чиновники".

**`LogicOperatorNode`** - нода содержащая один логический оператор. Стоит между субъектами или обстоятельствами в группе.

## Empty-ноды

Каждая конкретная нода имеет статическое поле `Empty` и свойство `IsEmpty`. Используется вместо `null` когда нода обязательна по типу но семантически отсутствует - например в результате diff-операции когда субъект был добавлен и у него нет предыдущей версии.

## Clone()

Каждая нода реализует `Clone()` объявленный абстрактным в базовом классе. Клонирование нужно при создании `LawStatement` из `BillStatement` - чтобы принятый закон хранил независимую копию тезиса и изменения в законопроекте не затрагивали уже принятый закон.
