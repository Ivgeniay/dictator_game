# Типы акторов

`Domain/Actors/Types/`

## Actor

Группа населения - анонимный коллектив объединённый общим субъектным типом. Моделирует давление на игрока со стороны социальных групп: народа, чиновников, бизнеса и других. Конкретные представители группы выражены через `Person`.

Хранит `ActorRelation` - групповое отношение к диктатору. Предоставляет фасадные методы для управления лояльностью и основаниями лояльности.

## Person

Конкретный именованный представитель группы населения. Моделирует индивида с которым игрок может взаимодействовать напрямую - подкупать, шантажировать, запугивать. Хранит личные грехи которые могут быть раскрыты в ходе расследования и личное отношение к диктатору через `PersonRelation`.

Реализует `IInvestigatable` - над персонажем можно проводить расследования.

## Dictator

Персонаж диктатора - доменная сущность отдельная от концепции игрока. Диктатор это персонаж в мире симуляции, игрок это тот кто им управляет. Хранит только идентификатор и имя на данном этапе.

## Organization

Абстрактный базовый класс для всех институциональных структур. Любая организация имеет владельца (`IOwner`) который может меняться - это механика национализации и приватизации свойственная диктатуре.

Хранит уровень лояльности и уровень коррупции. Доступ к изменению этих параметров - только через `OrganizationModerator`, единственный экземпляр которого создаётся один раз через `CreateSingle()`.

### Наследники Organization

`MediaOrganization` - СМИ. Может проводить журналистские расследования.

`PoliceOrganization` - полиция. Проводит официальные расследования.

`MilitaryOrganization` - армия. Источник угрозы военного переворота.

`ProsecutionOrganization` - прокуратура. Может возбуждать дела и при определённых условиях - против самого диктатора.

`BusinessOrganization` - бизнес. Источник финансирования и экономического влияния.

`NgoOrganization` - НКО. Проводит независимые расследования, создаёт международное давление.

## IOwner / IOwnable

`IOwner` - маркер сущности способной владеть организацией. Реализуется `Person` и `State`.

`IOwnable` - маркер организации имеющей владельца. Реализуется `Organization`. Метод `TransferOwnership` позволяет менять владельца в процессе игры.

## State

Синглтон представляющий государство как абстрактного владельца имущества. Используется когда организация принадлежит государству а не конкретной личности - парламент, министерства, муниципальные структуры.
